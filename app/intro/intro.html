Do you think you have what it takes to win a ticket to Devoxx?

Last year we learned machine learning, this year we're playing Tetris :)

Write an algorithm to play the games and get the highest scores. Upload your solution.

3 Win your ticket

Top two players win a ticket to Devoxx.

Contest

You will be given a list of 50 Tetris games in increasing difficulty (from 10 shapes to 10.000 shapes).
You will know in advance all the shapes in the game in the order they will come.
You will have to solve each game by specifying a move for each shape (the column where to place the shape and rotation to use).

Your goal is to score the highest score possible by taking advantage of the combo multipliers and planning your moves carefully.

You can solve the game using any programming languages or tools you want. All you have to do is upload a csv with your moves.
The Tetris engine used for scoring is provided only in js (see documentation), if you use another language you will have to write your own implementation.

Rules

Competition rules:
  • This is an individual competition. Teams are not allowed.
  • Use your company email for submissions
  • Any programming language, library or tool is allowed. (see example of js solution)
  • Any programming approach is allowed: if/else rules, greedy algorithms, decision trees, solution space exploring, machine learning etc.
  • You are allowed to upload as many submissions as you want (only your highest score is used for ranking)
  • Do not flood the server with submissions! Evaluate your solutions locally. (see js example or use the tetris-eval command line utility)
  • You must upload your solution code as a zip file before the end of the competition. It will be checked by the judges to avoid cheating.
  • Provide a short description of your solution in the zip file.
  • In case of multiple users with the same top score the judges will either provide a new set of games or will select the winners based on the solution source code and idea (the most technically challenging and comprehensive solution will win).

The Game

The Tetris game contains 7 shapes IJLOSTZ, which can be rotated 0,1,2 or 3 times.
The game is played on a grid of 10 columns and 20 rows.

Scoring

Each piece successfully placed on the grid will earn you 1 point.
Clearing rows will earn you an additional points (combo multipliers apply for clearing multiple rows at once):
  • 1 row: 100 points
  • 2 rows: 200 points
  • 3 rows: 400 points
  • 4 rows: 800 points

Games Input

The input file contains one game per line. Each character of a line represents a shape (one of IJLOSTZ) in the order they will come in the game.

Your goal is to place each piece on the Tetris grid and score the highest score possible.

The file will contain a total of 50 games with increasing difficulty.

  • LEVEL 0 - 10 trivial games of length 30 using 1 or 2 different shapes
  • LEVEL 1 - 10 easy games of length 50 using 3 or 4 different shapes
  • LEVEL 2 - 10 medium games of length 100 using all the 7 shapes
  • LEVEL 3 - 10 hard games of length 1000 using all the 7 shapes
  • LEVEL 4 - 10 extreme games of length 10000 using all the 7 shapes

Solution

The solution file is a csv with two columns: game and moves.

Example: 
game,moves
0,"0:0;2:0;4:0;6:0;8:0"
1,"0:0;2:1;5:0;6:1;8:1;4:2;0:0"

The game column contains the game id: from 0 to 49

The moves column contains your solution for a game.

A complete solution should have 51 lines (1 for the header and 1 for each of the 50 games from the input).
The first line should be the header: game,moves

The next 50 lines should each be a solution for one game (the order of the games does not matter).
A solution line has two comma separated values: the game id (from 0 to 49) and the moves string.
The csv values can be optionally enclosed in double quotes.

The moves string for a game should contain a move pair for each of the shapes in that game.
A move pair is represented as <column>:<rotation> and multiple moves are separated by semicolons.
Shapes are placed based on their top left corner.
The column represents the x position where to place the shape (after it is rotated!).
The rotation represents the number of rotations to apply to the shape before placing it.
For example 2:1; will rotate the shape once and place its top left corner on the column 2 of the grid. 


// games.txt
OOOOO
LSTSITS

//solution.csv
game,moves
0,"0:0;2:0;4:0;6:0;8:0"
1,"0:0;2:1;5:0;6:1;8:1;4:2;0:0"
