The Game

The Tetris game contains 7 shapes IJLOSTZ, which can be rotated 0,1,2 or 3 times.

Shape I (rotated 0 to 3 times)

Shape J (rotated 0 to 3 times)

Shape L (rotated 0 to 3 times)

Shape O (rotated 0 to 3 times)

Shape S (rotated 0 to 3 times)

Shape T (rotated 0 to 3 times)

Shape Z (rotated 0 to 3 times)

Games Input

The input file contains one game per line. Each character of a line represents a shape (one of IJLOSTZ) in the order they will come in the game.

Your goal is to place each piece on the Tetris grid and score the highest score possible.

The file will contain a total of 50 games with increasing difficulty.

  • LEVEL 0 - 10 trivial games of length 30 using 1 or 2 different shapes
  • LEVEL 1 - 10 easy games of length 50 using 3 or 4 different shapes
  • LEVEL 2 - 10 medium games of length 100 using all the 7 shapes
  • LEVEL 3 - 10 hard games of length 1000 using all the 7 shapes
  • LEVEL 4 - 10 extreme games of length 10000 using all the 7 shapes

OOOOO
LSTSITS

Solution

The solution file is a csv with two columns: game and moves.

Example: 
game,moves
0,"0:0;2:0;4:0;6:0;8:0"
1,"0:0;2:1;5:0;6:1;8:1;4:2;0:0"

The game column contains the game id: from 0 to 49

The moves column contains your solution for a game.

A complete solution should have 51 lines (1 for the header and 1 for each of the 50 games from the input).
The first line should be the header: game,moves

The next 50 lines should each be a solution for one game (the order of the games does not matter).
A solution line has two comma separated values: the game id (from 0 to 49) and the moves string.
The csv values can be optionally enclosed in double quotes.

The moves string for a game should contain a move pair for each of the shapes in that game.
A move pair is represented as <column>:<rotation> and multiple moves are separated by semicolons.
